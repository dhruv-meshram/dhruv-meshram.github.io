Trado — P2P Marketplace

A hyper-local peer-to-peer campus marketplace supporting real-time transaction negotiations, a mathematical trust score engine, and secure verification protocols.

Timeline

Fall 2025

Category

Full-Stack Web App

Codebase

GitHub Repository

Overview & Problem

P2P marketplaces on university campuses are often fragmented, relying on generic social media groups. These forums are plagued by spam listings, lack of trust between strangers, and inefficient coordination. Buyers and sellers frequently face safety concerns and high cancellation rates.

Trado addresses these inefficiencies by consolidating listings in a secure, campus-restricted web application. Features such as university email validation, mathematical trust metrics, and an integrated real-time chat with custom price proposals simplify transaction coordination.

Key Architectural Features

Trado's operation relies on three primary pillars of technology:

Socket.io Negotiation: Users do not just send text; they propose binding prices. The backend handles message payloads dynamically, rendering structured "Offer" cards directly inside the conversation stream.
Karma Trust Engine: Every profile features a rating generated by an algorithm. The score evaluates response speed, successful transaction count, and positive reviews, helping users gauge reliability.
Handshake Protocol: To complete a transaction securely, the seller receives a unique PIN code. Upon physical inspection of the item, the buyer inputs the PIN on their phone to finalize the sale and update profile histories.

Challenges & Learnings

A major engineering challenge was ensuring a smooth chat experience under spotty campus Wi-Fi. If connection dropped during a negotiation, messages could easily be lost. We resolved this by building a client-side database using IndexedDB.

Unsent messages are queued locally and marked with a pending state. A background synchronizer detects network restoration, flushing the local queue to the MongoDB server. This guarantees that negotiation histories remain accurate and complete, even with connection interruptions.

Tech Stack Detail

Next.js 14 & TypeScript

Provides a type-safe, component-driven framework supporting fast server-side rendering and static page optimization.

Socket.io & Node

Enables persistent bidirectional event channels between buyer and seller sockets for instant message updates.

MongoDB & Prisma

Flexible document schema for storing complex conversation histories, item states, and index optimization for geocoded queries.

NextAuth

Handles user sessions and restricts registration domains strictly to verified educational institute emails.
