Overview & Problem

Traditional project management software often abstracts away the mathematical complexity of task dependencies, leading to inefficient scheduling for large-scale projects. CPM Planner bridges this gap by integrating a dedicated C++ engine that performs rigorous Critical Path Method (CPM) calculations.

This ensures project timelines are mathematically optimized, identifying exactly which tasks are "critical" to the project's finish date and where "float" (slack time) can be utilized to manage resources effectively.

System Architecture

The project is built as a modular, three-tier system designed for performance and scalability:

C++ Mathematical Engine: A standalone, performance-optimized backend implementing Kahn’s algorithm for topological sorting. It performs forward and backward pass calculations to determine earliest/latest start times and critical paths for projects with thousands of tasks in milliseconds.
Interactive Workspace: A modern Next.js 15+ frontend providing real-time Gantt charts, interactive Directed Acyclic Graphs (DAG) via React Flow, and high-performance Kanban boards for task management.
Persistence Layer: A robust PostgreSQL database managed by Prisma ORM, handling multi-tenant workspaces, task versioning, and secure activity audit trails.
Integration Layer: A specialized Node.js service that serializes complex project graphs into JSON and invokes the native C++ engine via high-speed IPC (Inter-Process Communication), handling buffers up to 50MB for massive projects.

Challenges & Key Learnings

Engineering a full-stack mathematical optimization workspace presented several key architectural hurdles:

High-Volume Graph Processing: Optimizing the C++ engine for O(V+E) time complexity was essential for maintaining a responsive UI. The integration layer was designed to handle high-volume JSON serialization without blocking the main event loop.
Cycle Detection & Validation: To ensure mathematical validity, the system implements strict DAG (Directed Acyclic Graph) validation at both the database and engine levels, preventing circular dependencies that would cause deadlocks in scheduling.
Real-Time Visualization: Synchronizing the complex graph layout state (using Dagre algorithms) with React Flow components required advanced state management (Zustand) to keep the UI fluid even when managing deeply nested dependencies.
