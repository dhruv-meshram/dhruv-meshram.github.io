Gujarat Electricity Forecasting

A production-grade hourly electricity demand forecasting system built for the state grid of Gujarat, benchmarking gradient boosting against deep recurrent networks and optimized using ONNX Runtime.

Timeline

Winter 2025

Category

Time-Series & MLOps

Codebase

GitHub Repository

Overview & Problem

Grid operators must balance power generation and electrical demand in real time. Over-forecasting leads to costly fuel waste and excess carbon emissions, while under-forecasting compromises grid stability, triggering localized blackouts. Accuracy is essential: even a 1% deviation in regional grid load forecasting can result in massive operational cost overheads.

This system processes large-scale, multi-variable time-series data to deliver hourly predictions of electricity demand in Gujarat. Built using historical demand databases, complex weather logs, and state holiday calendars, the model equips utility operators with highly accurate 24-hour look-ahead forecasts.

Methodology & Feature Engineering

Raw time-series data was processed through a strict, leak-free pipeline with several custom feature layers:

Cyclic Time Encoding: Standard integer timestamps are transformed into cyclic coordinates (sine and cosine mappings of hour, weekday, and month) so models capture recurring temporal patterns natively.
Lag Features & Rolling Metrics: Includes historical load shifts (t-1 to t-24) alongside rolling statistical aggregates (mean, variance, min/max) computed over multiple time horizons.
Climate Interaction Factors: Captures non-linear dependencies between local temperature, relative humidity, and air conditioner usage by cross-multiplying raw weather inputs with cyclic summer indicators.

For model validation, we rejected simple cross-validation in favor of a 5-fold walk-forward backtesting strategy. This mirrors a real production cycle, ensuring that evaluation metrics are completely free from future data leakage.

Model Benchmarking & Deployment

We benchmarked two major time-series approaches: gradient-boosted decision trees (XGBoost) and deep recurrent networks (LSTM). While LSTM captured long-term sequences, XGBoost excelled at predicting sudden, sharp demand spikes driven by temperature fluctuations.

Our final system deploys a weighted ensemble model. To support sub-millisecond real-time queries, the model files are serialized to the Open Neural Network Exchange (ONNX) format. Using the ONNX Runtime C++ engine, average inference latencies dropped to 0.65ms, allowing seamless integration with regional SCADA monitoring systems.

Tech Stack Detail

XGBoost & PyTorch

Core forecasting model structures. PyTorch for long short-term memory recurrent networks; XGBoost for fast tabular regression.

ONNX Runtime

Deployment framework used to compile models into cross-platform binary representations, minimizing CPU latency.

Optuna

Autonomous Bayesian optimization framework used to find optimal neural layer configurations and learning rates.

Pandas & Numpy

High-performance data manipulation libraries used for feature engineering, interpolating missing points, and batch normalization.
